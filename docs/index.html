Patient Management System

Microservices-Based Healthcare Management Backend

Project Overview

This project is a microservices-based backend system for managing appointments, patients, doctors, and billing in a healthcare environment. It follows a modular, scalable design and uses different databases tailored to each service's needs.

Architecture

6 independent microservices with API Gateway

Containerization

Fully dockerized with Docker Compose

Event-Driven

Apache Kafka for service communication

Multi-Database

MySQL, PostgreSQL, and MongoDB

System Architecture

The system consists of the following microservices, each containerized via Docker:

Service Port Database Primary Function
API Gateway 4004 - Request routing and management
Patient Service 8080 MySQL Patient records management
Doctor Service 8083 MySQL Doctor profiles and specializations
Appointment Service 8084 PostgreSQL Appointment scheduling and validation
Billing Service 8081 PostgreSQL Payment processing and billing
Analytics Service 8082 MongoDB Data analytics and reporting

Microservices Details

👥

Patient Service

Database: MySQL | Port: 8080

Responsibilities: Managing patient records and validations

{ "id": Long, "name": String, "email": String, "phone": String, "registrationDate": Date }
👨‍⚕️

Doctor Service

Database: MySQL | Port: 8083

Responsibilities: Managing doctors and their specialties

{ "id": Long, "name": String, "specialization": ENUM, "phoneNumber": String, "email": String, "available": Boolean, "gender": ENUM }
📅

Appointment Service

Database: PostgreSQL | Port: 8084

Responsibilities: Creating appointments, verifying IDs, notifying other services

{ "patientId": Long, "doctorId": Long, "date": Date, "time": String, "serviceType": String }
💰

Billing Service

Database: PostgreSQL | Port: 8081

Responsibilities: Creating billing records after appointments

{ "patientId": Long, "appointmentId": Long, "paymentStatus": Boolean, "paymentType": ENUM }
📊

Analytics Service

Database: MongoDB | Port: 8082

Responsibilities: Storing event-based and analytical data (trends, usage stats)

🌐

API Gateway

Port: 4004

Responsibilities: Request routing, load balancing, and API management

Appointment Validation Flow

When a user tries to create an appointment, the system follows this validation process:

1
Request Received
The Appointment Service receives the appointment creation request
2
Patient Validation
Sends GET request to Patient Service to verify if the patientId exists
3
Doctor Validation
Sends GET request to Doctor Service to verify if the doctorId exists
4
Appointment Creation
If both validations pass, the appointment is created and stored
5
Billing Trigger
A billing record is automatically triggered and created

Key Features

  • Event-Driven Architecture: Uses Apache Kafka for loose coupling between services
  • Database Optimization: Each service uses the most appropriate database technology
  • Scalable Design: Independent services can be scaled based on demand
  • Docker Integration: Complete containerization for easy deployment
  • Real-time Validation: Cross-service validation ensures data integrity
  • Analytics Support: Built-in analytics for healthcare insights
